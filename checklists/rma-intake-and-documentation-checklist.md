# RMA Intake & Documentation Checklist

- [ ] Incoming board photographed from all angles before any power-on attempt
- [ ] Visible damage documented (U6/U11, RAM slots, ports, bent pins, etc.)
- [ ] Serial number and part number recorded
- [ ] Symptom clearly noted (No POST, No IPMI LED, etc.)
- [ ] Known-good components swapped for isolation (CPU, RAM, PSU)
- [ ] Full set of photos saved to 04-evidence/ folder
- [ ] RMA form completed with pattern notes if multiple units affected
- [ ] Board powered off and safely stored until RMA submission
